# 代码使用文档

本文档说明如何使用步态数据集完成基于 GaitSet 的步态分类训练与测试，并给出在 Windows `cmd.exe` 下可直接执行的命令。

配置文件：`configs/gaitset/anonymous_test.yaml`
---

## 1. 适用范围

这个配置主要用于以下场景：

- 模型：`GaitSet`
- 任务：三分类（`003 / 004 / 005`）
- 数据格式：`class`
- 训练策略：
  - 类别权重交叉熵
  - 不平衡采样 `ImbalancedSampler`
  - 特征层重采样 `feature_smote`
- 测试策略：固定标签映射 + `InferenceSampler`

- 训练/测试配置：`configs/gaitset/anonymous_test.yaml`
- 数据预处理脚本：`datasets/pretreatment.py`

## 3. 先决条件

在运行训练或测试前，请确认以下内容：

1. 已安装项目依赖，并且 PyTorch 环境可正常运行。
2. 数据已经预处理为 PKL 格式。
3. `configs/gaitset/anonymous_test.yaml` 中的路径已改为你的实际路径。
4. 测试所用的 checkpoint 已存在。

## 4. 配置文件关键项说明

### 4.1 数据配置 `data_cfg`

```yaml
data_cfg:
  dataset_name: MyClassification
  dataset_root: D:/Downloads/Anonymous-pkl
  dataset_partition: ./datasets/Anonymous/train_test.json
  num_workers: 2
  remove_no_gallery: false
  test_dataset_name: MyClassification
  data_structure: class
```

含义：

- `dataset_root`：预处理后 PKL 数据集根目录，需改成你的实际路径。
- `dataset_partition`：训练/测试划分文件。
- `data_structure: class`：表示数据按“类别名/身份ID/视角/帧”或“类别名/身份ID/样本/视角/帧”组织。
- `test_dataset_name`：测试时使用的数据集名。

### 4.2 测试配置 `evaluator_cfg`

```yaml
evaluator_cfg:
  restore_hint: ./output/MyClassification/GaitSet/GaitSet_Finetune_23/checkpoints/GaitSet_Finetune_23-06000.pt
  save_name: GaitSet_Finetune
  eval_func: evaluate_scoliosis
  label_mapping:
    "003": 0
    "004": 1
    "005": 2
```

含义：

- `restore_hint`：测试时加载的 checkpoint 路径。
- `eval_func`：分类评估函数，这里使用 `evaluate_scoliosis`。
- `label_mapping`：固定标签映射，训练和测试必须保持一致。
- `sampler`：测试时使用 `InferenceSampler`，`batch_size` 一般设为 1。

### 4.3 模型配置 `model_cfg`

- `model: GaitSet`
- `SeparateBNNecks.class_num: 3`：三分类头。
- `parts_num: 62`：与 GaitSet 特征分块设置有关。

### 4.4 损失配置 `loss_cfg`

当前配置包含：

- `TripletLoss`：权重为 `0.0`，即不作为主损失。
- `CrossEntropyLoss`：权重为 `1.0`，作为主要分类损失。
- `weight: [10.0, 0.556, 11]`：类别不平衡权重。
- `label_smooth: true`：启用标签平滑。

### 4.5 训练配置 `trainer_cfg`

```yaml
trainer_cfg:
  enable_float16: true
  restore_ckpt_strict: false
  optimizer_reset: true
  scheduler_reset: true
  save_name: Gaitset_Finetune_01
  total_iter: 6000
```

含义：

- `restore_ckpt_strict: false`：允许只加载匹配层，新增分类头可随机初始化。
- `optimizer_reset: true`：重新初始化优化器，适合微调新头。
- `scheduler_reset: true`：重新初始化学习率调度器。
- `save_name`：本次训练保存目录名。
- `total_iter`：总迭代次数。

此外，训练侧还启用了：

- `ImbalancedSampler`
- `TomekLinks`
- `feature_smote.enabled: true`
- `feature_smote.mode: smote`

## 5. 数据预处理

如果你的原始数据还没有转成 PKL，需要先执行预处理。

在仓库根目录下使用：

```cmd
cd /d D:\Project\OpenGait
python datasets\pretreatment.py -i "D:\Downloads\Anonymous18" -o "D:\Downloads\Anonymous-pkl" -n 4 --preserve_structure
```

参数说明：

- `-i`：输入原始数据目录
- `-o`：输出 PKL 数据目录
- `-n 4`：使用 4 个进程处理
- `--preserve_structure`：保留原始目录结构，适合当前匿名数据场景

> 如果你的原始路径不同，请把命令中的目录改成你自己的路径。

## 6. 训练方法

### 6.1 单次训练

在 Windows `cmd.exe` 中执行：

```cmd
cd /d D:\Project\OpenGait
python -m torch.distributed.launch --nproc_per_node=1 opengait/main.py --cfgs ./configs/gaitset/anonymous_test.yaml --phase train
```

说明：

- `--nproc_per_node=1`：单卡/单进程训练。
- `--cfgs`：指定配置文件。
- `--phase train`：进入训练模式。

### 6.2 按 `test.md` 的训练写法

`test.md` 里给出的单次训练命令是：

```cmd
python -m torch.distributed.launch --nproc_per_node=1 opengait/main.py --cfgs ./configs/gaitset/Anonymous_test.yaml --phase train
```

> 注意：仓库中实际文件名是 `configs/gaitset/anonymous_test.yaml`，Windows 下大小写通常不敏感，但建议统一使用仓库中的真实文件名。

### 6.3 训练输出

训练完成后，常见输出位置：

- checkpoint：`output/<Dataset>/<Model>/<RunName>/checkpoints/`
- 日志：`logs/` 下对应训练日志

当前配置的 `save_name` 是 `Gaitset_Finetune_01`，因此输出目录通常会包含这个名字。

## 8. 训练前必须检查的参数

建议每次开训前检查以下内容：

1. `data_cfg.dataset_root` 是否指向正确的 PKL 数据目录。
2. `data_cfg.dataset_partition` 是否存在。
3. `evaluator_cfg.restore_hint` 是否正确。
4. `trainer_cfg.label_mapping` 与 `evaluator_cfg.label_mapping` 是否一致。
5. `SeparateBNNecks.class_num` 是否等于类别数 `3`。
6. `loss_cfg.weight` 是否与你的数据类别分布匹配。
7. `save_name` 是否是你想要的实验名。

---

## 9. 常见问题

### 9.1 测试时找不到 checkpoint

检查：

- `restore_hint` 路径是否写错
- checkpoint 文件是否真的存在
- 是否和当前实验的 `save_name` 对应

### 9.2 分类头维度不匹配

检查：

- `SeparateBNNecks.class_num` 是否等于 3
- `label_mapping` 是否仍然是 `003/004/005`

### 9.3 训练不稳定或 batch 太小

检查：

- `trainer_cfg.sampler.batch_size` 是否合理
- 是否启用了 `ImbalancedSampler`
- `feature_smote` 在极小 batch 下是否造成样本不足

### 9.4 只想先做稳定基线

可以临时关闭部分不平衡策略：

- `trainer_cfg.sampler.type: CommonSampler`
- `feature_smote.enabled: false`
- 将 `CrossEntropyLoss.weight` 改为均匀权重
