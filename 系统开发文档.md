**系统开发文档 — MyGait 项目**

**项目概述**
- **项目名称**: 基于深度学习的步态识别研究框架。
- **主要功能**: 数据加载/采样、模型定义（GaitSet / Baseline / ResGCN 等）、训练/评估流水线、自动化批量实验脚本（leave-one-out）。
- **代码位置概览**: 项目主代码在 [opengait/](opengait/)，配置在 [configs/](configs/)，数据分区在 [datasets/](datasets/)，实验脚本在 [scripts/](scripts/)，输出写入 [output/](output/)。

**目录结构（关键项）**
- **opengait/**: 主代码
  - [opengait/main.py](opengait/main.py) — 程序入口，负责分布式初始化、加载配置、构建模型并启动训练/测试流程。
  - [opengait/data/](opengait/data/) — 数据集实现、采样、collate、transform。
  - [opengait/modeling/](opengait/modeling/) — 模型框架、models、backbones、losses、模块组件。
  - [opengait/evaluation/](opengait/evaluation/) — 评估指标、re-ranking 等实现。
  - [opengait/utils/](opengait/utils/) — 配置合并、DDP 封装、日志与工具函数。
- **configs/**: 配置目录
  - [configs/default.yaml](configs/default.yaml) — 全局默认配置，所有用户 cfg 与之合并。
  - [configs/gaitset/anonymous_test.yaml](configs/gaitset/anonymous_test.yaml) — 示例任务配置。
- **datasets/**: 数据分区与示例
  - [datasets/Anonymous/train_test.json](datasets/Anonymous/train_test.json) — TRAIN/TEST 分区文件（用于 leave-one-out）。
- **scripts/**: 自动化脚本
  - [scripts/leave_one_out_train.py](scripts/leave_one_out_train.py) — 批量训练（leave-one-out）。
  - [scripts/leave_one_out_test.py](scripts/leave_one_out_test.py) — 批量测试。
- **output/**: 训练/测试输出（checkpoints、summary、临时文件）。

**环境与依赖**
- **建议 Python**: 3.8+
- **主要依赖**: `torch`, `torchvision`, `numpy`, `opencv-python`, `PyYAML`, `tqdm`, `einops`（若使用 baseline），`scikit-learn` 与 `imbalanced-learn`（可选，用于 SMOTE/采样）。
- **GPU / 分布式**:
  - 程序使用 PyTorch Distributed，`main.py` 在启动时会验证 `torch.distributed.get_world_size() == torch.cuda.device_count()`；请使用 `torch.distributed.launch` 或 `torchrun` 启动，确保 `--nproc_per_node` 与可用 GPU 数匹配。
  - `main.py` 默认使用 backend `'gloo'`（可根据运行平台改为 `'nccl'`）。

**配置说明（关键字段）**
- **合并策略**: 用户指定的配置（`--cfgs`）会与 [configs/default.yaml](configs/default.yaml) 合并，用户 cfg 的字段会覆盖 default 中相同键。
- **常见字段**:
  - `data_cfg`: `dataset_name`, `dataset_root`, `dataset_partition`, `num_workers`, `data_structure`（`identity`/`class`）等。
  - `model_cfg`: `model`（模型名，映射到 `opengait/modeling/models` 中的类），以及 backbone/头部参数。
  - `trainer_cfg` / `evaluator_cfg`: `save_name`, `restore_hint`, `total_iter`, `save_iter`, `sampler`, `transform`, `enable_float16` 等。
  - `loss_cfg`: 支持单个 loss 或 list，包含 `type`、`loss_term_weight`、`margin` 等。
  - `optimizer_cfg` / `scheduler_cfg`: `solver`, `lr`, `weight_decay`, `scheduler` 等。

**数据格式与加载**
- 支持两种 `data_structure`：
  - `identity`: ID/view/frame 等结构（常用于步态数据）。
  - `class`: class/ID/view 层次（按类别组织的数据集）。
- 数据分区通过 `dataset_partition` 指向的 json（如 [datasets/Anonymous/train_test.json](datasets/Anonymous/train_test.json)）提供 `TRAIN_SET` / `TEST_SET`。
- 采样与 collate 由 [opengait/data/collate_fn.py](opengait/data/collate_fn.py) 与 [opengait/data/sampler.py](opengait/data/sampler.py) 控制，采样策略在 `trainer_cfg.sampler` 中配置（`fixed`/`unfixed`/`all` 等）。

**模型架构与模块**
- `opengait/modeling/` 是模型实现主目录：
  - [opengait/modeling/base_model.py](opengait/modeling/base_model.py) — 训练/测试框架、checkpoint 管理。
  - [opengait/modeling/models/gaitset.py](opengait/modeling/models/gaitset.py) — GaitSet 实现，含 HPP/BNNeck 等。
  - [opengait/modeling/models/baseline.py](opengait/modeling/models/baseline.py) — 基线实现。
  - Backbones 在 [opengait/modeling/backbones/](opengait/modeling/backbones/)（如 `resnet.py`, `resgcn.py`, `plain.py`）。
  - 损失函数在 [opengait/modeling/losses/](opengait/modeling/losses/)（`ce.py`, `triplet.py`, `supconloss.py`），由 `loss_aggregator.py` 聚合。

**训练与评估（如何运行）**
- 主入口：运行 [opengait/main.py](opengait/main.py)
  - 示例（单机多卡）:
    ```bash
    python -m torch.distributed.launch --nproc_per_node=4 opengait/main.py --cfgs ./configs/gaitset/anonymous_test.yaml --phase train --log_to_file
    ```
  - 测试（加载 checkpoint）:
    ```bash
    python -m torch.distributed.launch --nproc_per_node=1 opengait/main.py --cfgs ./configs/gaitset/anonymous_test.yaml --phase test --iter 6000
    ```
- 批量自动化（leave-one-out）:
  - 训练批量脚本:
    ```bash
    python scripts/leave_one_out_train.py
    ```
  - 测试批量脚本:
    ```bash
    python scripts/leave_one_out_test.py
    ```
- 输出约定：训练过程 checkpoint 存放在 `output/<dataset>/<model>/<save_name>/checkpoints/`，日志写入 `logs/` 或 `output/.../tmp/summary/`。

**常见问题与注意事项**
- **文件名一致性**: `scripts/leave_one_out_*` 中的示例配置名大小写需与 `configs/gaitset/` 下的文件一致（代码中发现 `Anonymous_test.yaml` vs `anonymous_test.yaml` 的名称不一致风险）。
- **分布式 backend**: `main.py` 使用 `'gloo'`，在 Linux + NVIDIA GPU 环境更常用 `nccl`，请根据运行平台调整。
- **SMOTE / 采样**: 如使用 `feature_smote`，请确保安装 `imbalanced-learn` 或所需库。
- **平台兼容**: 部分脚本或 shell 命令在 Windows 下不可用，请在文档中注明平台限制并建议在 Linux 环境运行训练/评估任务。

**开发规范与贡献**
- 新模型请放入 `opengait/modeling/models/` 并在 `modeling/__init__.py` 或 `base_model` 的模型工厂中注册。
- 新 backbone 放入 `opengait/modeling/backbones/` 并提供 `get_backbone` 可调用接口。
- 配置文件：新增配置请放 `configs/<task>/` 下，并使用 `configs/default.yaml` 做默认字段继承。


**附录：快速开始（最小示例）**
1. 安装依赖（示例）:
   ```bash
   pip install torch torchvision pyyaml numpy opencv-python tqdm einops scikit-learn imbalanced-learn
   ```
2. 准备数据并配置 `configs/gaitset/anonymous_test.yaml` 中的 `data_cfg.dataset_root` 与 `dataset_partition`。
3. 单卡调试运行：
   ```bash
   python -m torch.distributed.launch --nproc_per_node=1 opengait/main.py --cfgs ./configs/gaitset/anonymous_test.yaml --phase train
   ```
